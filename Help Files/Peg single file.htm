Display screen:  Pegaesys uses a CRT monitor VGA or Super VGA.  The display includes the following elements:

Line 1:  Caption bar, required by Windows, serves as a title and as a report line.

Line 2:  MenuBar, required by Windows, accessed using the <Alt> key or mouse.

Status line:  The line at the top of the drawing area for standing reports, transparent to Graphic functions.  Contents:

Segment count  Trap buffer count   Pen color  Line style  Text height  Scale  Screen ratio  Tab length,  angle

Toolbar line:  A transparent pseudo bar at screen bottom serves as a menu for the numeric commands in the current mode.  Press the number; dont click the screen.  When the mode changes, the toolbar changes to indicates the new commands.  Pending operations are indicated by a different color.

Cursor is unique for each GIN mode for instant recognition.

Odometer at screen top right displays the cursor position relative to the working origin.

The display is as uncluttered as possible, maximizing the area available for drawing.  The transparent reports can be turned off using F2, F3 and F4,  see MenuBar Setup.

PegAEsys File Structure

Graphic Files store data used to represent drawings. This data creates primitive drawing elements like alphabet letters create words.  Primitive drawing elements are points, lines, circles, arcs, curves, ellipses, and text.   Primitive elements may be combined into more complex segments, like words make a phrase.   Segments can be combined to form even more complex segments such as reusable blocks or compressed segments like words and phrases make a sentence.   Segments can be organized into Layers like sentences make a paragraph.   Plans  consist of  views and details on sheets bound into a set of drawings like paragraphs on pages bound in a book.

PegAEsys Graphic Files can be like that, for compatability with other CAD systems.  Internal blocks and layers are supported, but since they are bound in a single file, access is limited to one terminal at a time.  PegAEsys can put multiple simple drawings called tracings together to form sheets like a loose-leaf book.   What this means is that more than one person can work on a sheet at the same time.  

Since 3 degrees of access (Open, Map, and View) are supported, you can open an included tracing, map or view the remainder of the sheet. PegAEsys freely moves data between internal layers and external files.  Even if you start with a bound drawing, you can Melt it into several files, work on them simultaneously, and  Fuse back into a single file.  Usually they are plotted while separated and only recompiled when they are archived.

Pegaesys is 3-D CADD

Peg16 is full featured and runs under Microsoft Windows 3.1 or later, on IBM 386sx compatable PCs or better with VGA display or better, and can use a Local Area Network. Faster CPU, coprocessor, more memory, and a bigger screen improves performance. 

Peg16 is the 16-bit evaluation version of Pegaesys.  It is compiled for minimum size to run under any current  Windows on the widest range of platforms.  Peg32 is the commercial version.  It is compiled for maximum speed on Windows NT and Windows 95.    

Pegaesys uses the Microsoft  Windows family of operating platforms.  Windows protocol is used to a point, but IO is unique for speed and accuracy. 

PegAESys is  complex software, and uses many commands:

Pegaesys divides commands into Modal and Global functions.

Mode describes the general activity, like Drawing, Dimensioning, Selecting, Annotating ...

Choose any Mode, and the numeric keypad provides 10 commands for that mode.

Global describes a common activity needed for all the modes, like Snaps, Pick, Delete, Move, ....

Global command keys use the typewriter keyboard and are available regardless of the Mode. 

Pull down Menus are provided for all commands, but high speed IO is achieved using short-cut keys.

Pegaesys is Lean:  Peg##.exe,  Help,  Data Files,  and demo drawings fit on a 3.5 floppy disk.

You can run Pegaesys from its floppy disk, albeit slowly.

Q   Segment Snap (Engage a segment)

Move the cursor near the target segment

      (but not beyond its endpoint) .

Press  Q to  snap to and engages the nearest segment. 

The segment blinks continuously to confirm engagement. 

The length and angle of an engaged LINE segment are reported.

Only 1 segment may be engaged at a time.

If the cursor is  beyond the endpoint of the segment, the segment is ignored.

Pressing  Q  sequentially  snaps the cursor to the opposite end of the segment. 

Sequential operation of  Q  never moves the cursor off the engaged segment.

Text segments are ignored.

Polygons are treated as a series of contiguous vectors.

Press SpaceBar to disengage the segment.

[Amber demo]

Move the cursor near the selected node.

Press   A  to snap to the node and engages the segment.

The segment blinks to confirm engagement.

The length and angle of an engaged LINE segment are reported.

Only 1 segment may be engaged at a time.

This operation will always move the cursor off of the previously engaged segment if another node exists within the pick radius.  If more than 2 segments exist at a node, sequential  A   operations process the endpoints in rotation.

Text segments are ignored.

Polygons are treated as a series of contiguous vectors.

Press SpaceBar to disengage the segment.

Wrk_origin        to reset the working origin.

Notes:

Pivot            same as pressing 1 in Gin Mode: Edit.

Wrk_origin        is the point from which all position reports annunciate. 

Where X Y Z cursor position is reported, it means distance from the  working origin.

View_ref        is the center of the screen. 

If you reset the view reference point and then compose a new view, the new view will generate with the view reference point at the center of the screen.

1 2  3  4... 9        are stake numbers which you set in a session with the intent of returning to later. 

They can be set in overview to pre select pan repaints.  They can also be used to recover previous cursor position.

Abs_origin        is the registration point for the entire drawing.  Resetting this parameter in effect moves every element of the drawing.
